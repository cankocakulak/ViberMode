# Product to Spec

Read and follow the full workflow instructions at `../viber-mode/packs/vibermode/workflows/product-to-spec.md`.

Before starting:
1. Derive or confirm `project-name`
2. Resolve the target repo root for `docs/[project-name]/`
3. Prefer existing upstream artifacts when they already exist
4. Stop only when `spec-review.md` reaches `APPROVED` or `BLOCKED`

Primary artifact set:
- `docs/[project-name]/brainstorm.md`
- `docs/[project-name]/prd.md`
- `docs/[project-name]/ux.md`
- `docs/[project-name]/stories.md`
- `docs/[project-name]/spec-review.md`
